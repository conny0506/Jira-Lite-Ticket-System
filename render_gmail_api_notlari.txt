Render ENV update notes (Gmail API)

1) Required variables
EMAIL_PROVIDER=gmail_api
GMAIL_CLIENT_ID=
GMAIL_CLIENT_SECRET=
GMAIL_REFRESH_TOKEN=
GMAIL_SENDER_EMAIL=your-verified-sender@your-domain.com
EMAIL_HTTP_TIMEOUT_MS=15000
PASSWORD_RESET_URL_BASE=https://jira-lite-ticket-system-web.vercel.app/reset-password

2) Keep (already in use)
WEB_ORIGIN=https://jira-lite-ticket-system-web.vercel.app
JWT_SECRET=...
DATABASE_URL=...
REDIS_URL=...

3) Optional / unused
- When `EMAIL_PROVIDER=gmail_api`, `SMTP_*` values are ignored.
- When `EMAIL_PROVIDER=gmail_api`, `RESEND_*` values are ignored.
- You can remove `SMTP_*` and `RESEND_*` env vars (optional).

4) Google Console side
- Create OAuth consent screen.
- Enable Gmail API.
- Create OAuth Client (Web app).
- Generate refresh token (offline access).
- Add the `GMAIL_*` values to Render env settings.

5) Final step
- Redeploy/restart the Render API service.
